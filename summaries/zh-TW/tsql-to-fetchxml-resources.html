01SQL 4 CDS

首選開源NuGet 可嵌入TDS endpoint
作者 Mark Carrington
GitHub github.com/MarkMpn/Sql4Cds
NuGet MarkMpn.SQL4CDS
官網 markcarrington.dev/sql-4-cds

它能做什麼

讓你用標準 SQL 語法查詢與操作 Dataverse / D365 的資料與 metadata，支援 SELECT / INSERT / UPDATE / DELETE。在可能時，查詢會被轉成 FetchXML —— 因此你可以用熟悉的 SQL 產生 FetchXML，再用於 plugin 或整合。

為什麼適合 NL2SQL / TDS 場景

  • 核心 engine 是獨立 NuGet 套件，可直接嵌入程式或 pipeline，不必透過 GUI ——適合做為 eval pipeline 的程式化元件。
  • 同時提供 .NET Framework 4.6.2+ 與 .NET Core 3.1+ 組件。
  • engine 會把所有「FetchXML 有直接對應」的 SQL 語法轉成 FetchXML；無對應的功能（calculated fields、HAVING 等）則盡量把可下推的 filter 放進 FetchXML，其餘在記憶體中計算。
  • 可直接走 TDS endpoint：connection 屬性可指定在可行時用 preview TDS Endpoint 執行 SELECT。
  • 內建大量 FetchXML / TDS endpoint 不直接支援的 SQL 功能。

進階能力

  • Metadata 查詢：把 SQL 轉成 RetrieveMetadataChangesRequest / RetrieveAllOptionSetsRequest。
  • CTE 支援（v8.0.0 起）：non-recursive 展開成 subquery 以相容 TDS Endpoint；recursive 在可能時轉成 hierarchical FetchXML filter。
  • JSON：OPENJSON、JSON_QUERY、ISJSON。
  • 效能：RetrieveTotalRecordCountRequest 加速 SELECT count(*)、自動分頁、繞過 AggregateQueryRecordLimit。

產生的 FetchXML 中會帶上 xmlns:generator="MarkMpn.SQL4CDS" 命名空間並盡量內含完整 filter，例如：

<fetch xmlns:generator="MarkMpn.SQL4CDS">
  <entity name="contact">
    <attribute name="firstname" />
    <attribute name="lastname" />
    <attribute name="contactid" />
    <filter>
      <condition attribute="statecode" operator="eq" value="0" />
    </filter>
  </entity>
</fetch>

取得方式

  • XrmToolBox 外掛：xrmtoolbox.com/plugins/MarkMpn.SQL4CDS
  • SSMS / Azure Data Studio 擴充
  • NuGet 程式化嵌入：MarkMpn.SQL4CDS
部分查詢無法純靠 FetchXML 執行，這類會出現警告，表示需要 SQL 4 CDS 額外的記憶體內處理。這個「可否完全下推」的訊號，剛好可拿來當 eval 指標。

02SQL2FetchXML

線上服務KingswaySoft免安裝
網址 sql2fetchxml.com
Help sql2fetchxml.com/Help.aspx

貼上 SQL 點 convert 即得 FetchXML，適合快速試手。支援 SELECT FROM WHERE AND OR LIKE GROUP BY ORDER BY AS TOP DISTINCT NOLOCK INNER JOIN LEFT OUTER JOIN ON IN 與聚合 COUNT/MAX/MIN/AVG/SUM，並會自動移除 Filtered 前綴。

運算子表示法

FetchXML 專屬運算子用去掉 dash 的 SQL 函式表示：next-x-weeks → nextxweeks(n)、today → today()、eq-userid → equserid()。若運算子本身是 SQL 關鍵字（如 on、in）需加 x 前綴：

SELECT * FROM contact
WHERE createdon = xon('2011-01-11T00:00:00')
已知限制：對括號、巢狀 select 處理不佳；大小寫偶有出錯；無法處理 NOT IN / NOT EXISTS / EXCEPT（多半其實是 FetchXML 本身限制）。嚴謹評估建議以 SQL 4 CDS 為主，本工具僅作快速參照、勿當 ground truth。

03FetchXmlToSql Custom API

官方內建未文件化反向

方向與你需求相反（FetchXML → SQL），但屬官方途徑。透過 Dataverse 內建但未文件化的 Custom API Function FetchXMLToSql：

GET https://{your_environment}.api.crm.dynamics.com
    /api/data/v9.2/FetchXMLToSQL(FetchXml=@p1)
    ?@p1='{Your_FetchXML}'

也可在 XrmToolBox 用 Custom API Tester 呼叫。常見用途是 virtual table 指向 SQL Server 時把 FetchXML 轉回 SQL。

⚠️ 因屬未文件化功能，無保證持續可用，正式產品不建議依賴。

04FetchXML Builder

開源XrmToolBox多語法輸出
外掛 xrmtoolbox.com/plugins/Cinteros.Xrm.FetchXmlBuilder
官網 fetchxmlbuilder.com

對 Dataverse / Power Apps / D365 建立與執行 FetchXML。對本主題的價值在於能在多種語法間互轉：產生 C# QueryExpression、WebAPI / OData query string、Power Automate List Records、SQL / JavaScript / C# stub。可與 SQL 4 CDS 互通 —— 在 SQL 4 CDS 寫 SQL → 丟到 Builder 編輯，反之亦可。

05FetchXML Studio

開源Power Platform ToolBox視覺化
GitHub github.com/mohsinonxrm/pptb-mxrm-fetchxml-studio

Power Platform ToolBox 生態系的開源視覺化 FetchXML 建構器。主要用途是編寫 FetchXML，但整合了 §03 提到的未文件化 FetchXMLToSQL API，提供 SQL ↔ FetchXML 雙向預覽，可當作語法對照時的另一個快速工具。

06官方 SDK 範例：FetchXML ↔ QueryExpression

開源範例PowerApps-Samples

文件：learn.microsoft.com — Convert queries sample

示範用 QueryExpressionToFetchXmlRequest 與 FetchXmlToQueryExpressionRequest 雙向轉換。雖非 T-SQL→FetchXML，但若 pipeline 已用 QueryExpression 作中介層，這是官方支援的轉換路徑。

→給 Dataverse NL2SQL / TDS 的建議

  1. 程式化嵌入：直接用 MarkMpn.SQL4CDS engine NuGet，把模型產生的 T-SQL 餵進去轉 FetchXML，並比對是否「可完全下推」(fully foldable) 作為一個 eval 指標。
  2. 可下推性是好訊號：能完全轉成 FetchXML，代表查詢落在 Dataverse 原生能力內；需要記憶體處理的部分可標記為「超出 FetchXML / 可能超出 TDS endpoint」案例。
  3. TDS endpoint 對齊：SQL 4 CDS 的 connection 屬性可切換是否走 TDS endpoint，方便比較 FetchXML 路徑與 TDS 路徑的行為差異（呼應你先前測 OFFSET / EXCEPT / CTE 支援度的工作）。
  4. 線上快查：臨時想看某段 SQL 的 FetchXML 長相時用 SQL2FetchXML，但別當成 ground truth。
